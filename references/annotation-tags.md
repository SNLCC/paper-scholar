# 批注功能标签库与 L1/L2/L3 体系

## 核心理念

不是穷尽式覆盖所有批注类型，而是提供初始标签库，随使用不断丰富。
同时防止同功能标签多次出现（如"概述"和"总体概括"应合并）。

## L1/L2/L3 三级批注体系

### L1：标签订义（描述层）

根据论文内容直接打标，回答"这段文本在做什么功能"。

### L2：类型归纳（模式层）

从多次 L1 标注中归纳出该用户的批注类型偏好。

### L3：风格提取（迁移层）

即使用户不批注，也能以用户的方式对论文进行批注。

## 标签管理原则

### 防膨胀规则

1. **同义合并**：新标签与已有标签语义相似度 ≥ 0.8 → 合并
2. **频率门槛**：某标签使用次数 < 3 次且存在 > 3 个月 → 建议合并到上级标签
3. **命名规范**：统一使用"名词短语"格式（如"合理性解释"而非"解释合理性"）

### 扩展规则

1. 新标签不与已有 21 个标签中的任一个重复
2. 新标签描述了一种已有标签无法覆盖的功能
3. 新标签至少在 3 次批注中被独立使用

## L3：风格提取（无批注时的自主批注）

当用户批注数据积累到 ≥ 20 篇后，skill 应能在未阅读用户批注的情况下，以用户的方式对论文进行批注：

1. 按用户的标签分布分配分析精力
2. 按用户的粒度决定标注密度
3. 使用用户常用的标签类型标注
4. 按用户的颜色体系分类输出
