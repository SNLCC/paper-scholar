# 批注方式学习

## 核心理念

目标不是"读取用户的批注"，而是**学习用户批注的方式**——用户的批注风格、逻辑层次、精细度、关注焦点——使得即使**用户不批注**时，skill 也能以用户的方式做出高质量批注。

---

## 三级学习模型

| 级别 | 名称 | 说明 |
|------|------|------|
| L1 | **读取批注内容** | 读取 Zotero 中的高亮和评注，理解用户关注什么 |
| L2 | **学习批注方式** | 从批注中归纳用户的批注风格、逻辑层次、精细度标准 |
| L3 | **迁移批注能力** | 即使论文无批注，也能用学到的"用户的方式"进行批注 |

目前大多数"批注学习"工具只停留在 L1。**本 skill 的目标是达到 L3**。

---

## 从批注中学习的维度

### 维度一：批注精细度

用户批注的颗粒度反映了用户期望的分析深度：

| 精细度级别 | 表现 | 对 skill 的影响 |
|-----------|------|----------------|
| **粗粒度** | 只标段落，不标句子 | 精读时段落级详细、句级可适当精简 |
| **中粒度** | 标关键句子 | 精读时每段选标 1-2 句重点 |
| **细粒度** | 逐句标注，部分有评注 | 精读时逐句分析，这是默认模式 |
| **极细粒度** | 逐句标注 + 每句有评注 | 精读时每句都需包含作者意图分析 |

**学习方式**：统计用户批注的密度（批注数 / 总句数 × 100%）。

```
用户过去 10 篇论文的批注密度：8.5%
→ 用户属于"选标关键句"类型
→ 精读时在细粒度分析基础上，优先呈现与用户风格匹配的标注
```

### 维度二：关注焦点

用户批注在不同章节的分布揭示了关注重心：

```json
{
  "user_id": "default",
  "focus_profile": {
    "introduction": {"ratio": 0.15, "count": 12},
    "theory_framework": {"ratio": 0.35, "count": 28},
    "methodology": {"ratio": 0.10, "count": 8},
    "analysis_findings": {"ratio": 0.25, "count": 20},
    "discussion": {"ratio": 0.15, "count": 12}
  },
  "evidence": "基于 15 篇论文的 80 条批注统计"
}
```

### 维度三：批注类型偏好

用户批注中隐含的判断类型：

| 批注类型 | 识别方式 | 示例 |
|---------|---------|------|
| **认可型** | 高亮无评注，或评注含"好""精辟""重要" | 高亮一段话 |
| **质疑型** | 评注含问号、"存疑""为什么""不对" | "这个假设有问题？" |
| **关联型** | 评注含"参见""对比""类似于" | "与 Smith(2020)观点一致" |
| **延伸型** | 评注含"可以研究""这个方向""有意思" | "这个缺口可以跟我的选题结合" |
| **方法型** | 评注含"方法""数据""样本" | "DID 方法，注意平行趋势检验" |

### 维度四：颜色编码体系

如果用户有系统性使用批注颜色：

```
黄色（#ffd400）→ 核心观点（60%使用率）
红色（#ff6666）→ 质疑/问题（20%使用率）
蓝色（#2ea8e5）→ 可引用/方法（15%使用率）
绿色（#5fb236）→ 待查/延伸（5%使用率）
```

**重要**：颜色含义**不是预设的**，而是从用户的批注模式中统计推断的。如果用户在红色评注中写"好方法"，那红色可能不是质疑的意思。

## .learnings/ 机制：偏差学习

### 核心理念

当 skill 的分析与用户的批注存在差异时，不是"skill 要改正"，也不是"用户错了"，而是将偏差案例记录下来，供后续迭代优化。

**原则**："小研分析可能更优"——不强行对齐用户批注。用户的批注是学习素材，而非标准答案。

### 偏差记录流程

```
skill 完成精读分析
    │
    ├─ 读取用户 Zotero 批注
    │
    ├─ 对比：skill 的分析 vs 用户批注
    │
    ├─ 差异识别：
    │   ├─ 用户标注了某句，skill 未重点分析 → skill 漏标
    │   ├─ skill 标注了某句，用户未标注 → 可能非用户关注点
    │   └─ 用户评注与 skill 分析结论不同 → 视角差异，值得记录
    │
    └─ 写入 .learnings/ 目录
```

### 偏差记录格式

每篇论文的对比结果存入 `.learnings/{paper_id}.json`：

```json
{
  "paper_id": "ABC123",
  "title": "论文标题",
  "comparison_summary": {
    "skill_annotations": 45,
    "user_annotations": 12,
    "overlap": 8,
    "skill_only": 37,
    "user_only": 4
  },
  "notable_discrepancies": [
    {
      "location": "p.3, para 2, sentence 1",
      "skill_analysis": {
        "function": "建立研究合法性",
        "author_intent": "让读者认可研究价值",
        "template": "While [prior work]..."
      },
      "user_annotation": {
        "text": "这个gap表述很清晰",
        "type": "认可型",
        "color": "#ffd400"
      },
      "insight": "用户特别关注 gap 陈述的清晰度——精读时对缺口声明的分析应更详细"
    }
  ],
  "learning_points": [
    "用户偏好：喜欢在理论框架部分做详细批注",
    "用户风格：每段只标最核心的1-2句",
    "用户关注：方法论的稳健性讨论"
  ]
}
```

### 偏差学习的聚合

定期（每完成 5 篇对比后），将 `.learnings/` 中的各篇记录聚合成一份用户画像更新：

```bash
# 学习聚合的结果写入 annotation-learning.md 的用户画像部分
```

聚合结果示例：

```
基于 12 篇论文的批注对比分析，已识别出以下稳定模式：
1. 用户批注精细度：中粒度（每段 1-2 句，密度约 6%）
2. 核心关注焦点：理论框架 (35%) 和论证分析 (25%)
3. 批注类型偏好：认可型 (40%)、质疑型 (25%)、延伸型 (20%)
4. 颜色模式：黄色=核心观点，红色=质疑/待讨论
5. 典型差异模式：用户比 skill 更关注"理论之间的对话关系"
```

## L3 目标：无批注时的自主批注

当用户批注数据积累到一定程度（建议 ≥ 20 篇），skill 应能在未阅读用户批注的情况下，以**用户的方式**对论文进行批注：

```
新论文（无用户批注）
    │
    ▼
调取用户画像（从 .learnings/ 聚合）
    │
    ├─ 按用户的焦点分布分配分析精力
    ├─ 按用户的精细度决定输出粒度
    ├─ 按用户的批注类型标注每句
    └─ 按用户的颜色体系分类输出
    │
    ▼
输出符合用户风格的标注版本
    │
    ▼
用户审阅 → 认可或修正 → 反馈进入 .learnings/ 循环
```

## 目录结构

```
paper-scholar/
├── .learnings/                    # 偏差学习记录（自动生成）
│   ├── paper_ABC123.json
│   ├── paper_DEF456.json
│   └── profile.json              # 聚合的用户画像
├── references/
│   └── annotation-learning.md     # 本文档
```

`.learnings/` 目录应纳入 `.gitignore`（因为它包含用户个性化的分析偏好，不同用户不同）。
