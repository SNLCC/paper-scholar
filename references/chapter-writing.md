# 各章节写作指导

本文档提供基于模型的论文章节写作指导方法论。

**重要提示**：本文档提供的是写作的通用策略框架，而非固定模板。具体写作建议应结合模型库中同类论文的积累结果。

---

## 写作指导的生成逻辑

```python
def generate_writing_guide(model, user_topic):

    # 2. 为每个章节提取典型的论证步（rhetorical moves）
    # 3. 附上置信度标识

```

即：**模型提供结构骨架，用户主题填充内容，两者结合生成写作指导。**

## 通用章节写作策略

以下是各类论文章节的常见写作策略框架。每个策略都附带**应当问自己的问题**，帮助用户判断该策略是否适合自己的论文。

### 引言 / Introduction

**核心任务**：让读者认可该研究值得关注，并清晰理解论文要做什么。

**常见推进路线**：

| 步 | 策略 | 要思考的问题 |
|----|------|------------|
| 开场 | 用一个现象/问题/统计数据或引人深思的设问引入领域 | 我的读者为什么要关心这个话题？ |
| 背景 | 简要勾勒研究领域的关键脉络 | 哪些已有的研究构建了本领域的基础？ |
| 聚焦 | 从宽泛背景聚焦到具体研究问题 | 在已有研究中，具体哪个问题尚未被充分解答？ |
| 缺口 | 明确指出既有研究的不足或未解问题 | 为什么这个缺口是重要的？不填补它会怎样？ |
| 回应 | 宣布本文的研究目的/假设/方法 | 本文准备如何填补这个缺口？将采用什么路径？ |
| 预览 | 预告论文的组织结构 | 读者可以期待接下来的章节如何展开？ |

**常见陷阱**：
- 开场过于宽泛，没有迅速聚焦（"在当今瞬息万变的世界……"）
- 缺口叙述太弱，没有说服读者为什么这个缺失是重要的
- 承诺了引言中的内容但在正文中没有兑现

### 文献综述 / Literature Review

**核心任务**：展示作者对既有研究的全面把握，同时为本文的贡献建立舞台。

**常见推进路线**：

| 策略 | 说明 |
|------|------|
| **主题式组织** | 按研究主题/流派组织，而非按时间顺序罗列 |
| **对话式叙述** | 让不同研究之间形成"对话"，而非"A说了X，B说了Y"的清单 |
| **批判性整合** | 对每个流派做出评价，指出其优势和局限 |
| **缺口凸显** | 在每个主要主题的结尾，指出还有哪些未尽之处 |
| **通向本文的桥梁** | 综述的结尾明确指向本文的研究空间 |

**要点**：
- 文献综述不是读书笔记。它的目的是论证为什么本文的研究是有必要的
- 每段应有自己的论点，而不只是文献内容的摘要
- "Gap"的揭示应当是文献综述内在的论证结果，而非强加的外部判断

### 方法 / Methodology

**核心任务**：让读者相信本文的研究设计是严谨的、适合回答研究问题的。

**常见要素**：

| 要素 | 说明 |
|------|------|
| 研究方法的选择 | 为什么选这个而非其他方法 |
| 数据来源 | 数据从哪里来，为何选择这个来源 |
| 分析策略 | 如何处理数据/文本，分析步骤是什么 |
| 研究局限 | 方法本身固有的局限（坦诚对待） |

**要点**：
- 方法章节不是操作手册。重点是论证选择背后的理由
- 对不同性质的论文，方法的含义不同——实证研究有数据和分析方法，理论论文有概念分析方法，文本分析有关注和解释策略

### 分析 / 论证展开 / Findings

**核心任务**：呈现作者的核心论证或研究发现。

**要点**：
- 论证展开应当与理论框架呼应——框架中提出的概念如何在分析中被使用
- 证据的呈现应当有层次：核心发现优先，辅助发现其次
- 每个分析段落应当包含原始材料（引文/数据）和分析（解释材料的意义）
- 分析不是描述，而是论证——"这个材料说明了什么"比"这个材料是什么"重要

### 讨论 / Discussion

**核心任务**：解释研究发现的意义，将其放回更广阔的学术语境。

| 策略 | 说明 |
|------|------|
| 回扣引言 | 回到引言提出的问题，说明研究发现了什么 |
| 与既有研究对话 | 本文的发现与之前综述过的文献一致还是冲突？为什么？ |
| 理论贡献 | 本文的研究如何推动了理论的发展 |
| 实践意义 | 研究发现对现实世界有何意义 |
| 局限与展望 | 坦诚面对不足，指出未来方向 |

**陷阱**：
- 简单重复分析章节的内容，而没有加入新的阐释
- 回避与既有研究不一致的发现
- 局限部分过于谦虚，削弱了全文的可信度

### 结论 / Conclusion

**核心任务**：给读者留下清晰、持久的结构性印象。

**要点**：
- 总结核心贡献（2-3点即可）
- 强调研究的意义（理论 + 实践）
- 前瞻性展望（3-5个未来方向）
- 以一个有力的结束句收尾

## 写作指导的输出格式

每次生成写作指导时，按以下格式输出：

```markdown
## [章节名称] 写作指导

**核心任务**：[一句话说明该章节要达成什么]

**建议推进路线**：[基于模型推荐的论证流]

**各步详解**：

### 步1：[名称]
[策略说明]
示例句式：
> [模型中的可迁移句式模板，置信度 XX%]

### 步2：[名称]
[策略说明]

...

**常见陷阱**：
- [陷阱1]
- [陷阱2]

**置信度说明**：本建议基于 [n] 篇同类论文分析，模型置信度 [0.XX]
```

## 模型不完善时的处理

如果所选模型的置信度较低（< 0.3），或用户的问题涉及一个模型库中尚未积累的新论文类型，写作指导应采用以下策略：

1. **坦诚告知用户**：这个类型目前仅有少量样本，以下建议供参考
2. **提供通用策略**：回归到本文档中的通用写作策略框架
3. **建议先精读更多同类论文**：推荐用户先精读 3-5 篇目标期刊/类型的论文后再开始写作
