# 完整工作流闭环

## 10 阶段全流程

```
阶段0：获取论文（PDF/Zotero/WebDAV）
    ↓
阶段1：质量评分（期刊30% + 作者15% + 论文特征35% + 时效20%）
    ↓ ← 动态调整（可升级/降级阅读深度）
阶段2：类型判断（学术述评/实证研究/理论建构/文本分析/…）
    ↓
阶段3：100% 覆盖率精读（8条规范，三层分析）
    ↓
阶段4：批注对比校准（双向语义对齐 + 学习模式识别）
    ↓
阶段5：理解验证（转述检验 + 反例检验 + 衰减预判）
    ↓ ← 通过
阶段6：模型更新（先查后写 + 去重合并 + 置信度升级 + 整洁度检查）
    ↓
阶段7：选题分析与论文评分（时代局限 + 研究关联 + 引用价值）
    ↓
阶段8：写作触发（查模型 → 推荐结构 → 生成指导 → 置信度说明）
    ↓
阶段9：复现触发（记忆浮现：从已读论文中提取骨架）
```

## 与旧版 8 阶段的区别

| 旧版（v1.1.0） | 新版（v1.2.0+） |
|--------------|--------------|
| 阶段4：批注校对（单向罗列） | 阶段4：批注对比校准（双向语义对齐） |
| 阶段5：理解验证（仅时效局限） | 阶段5：理解验证（三项检验） |
| 阶段6：模型更新（简单合并） | 阶段6：模型更新（先查后写+规则） |
| 无 | 阶段9：复现触发（记忆浮现） |

## 各阶段的互锁关系

- **阶段5（理解验证）** 是阶段6（模型更新）的守门人
- **阶段4（批注对比）** 的反哺数据进入 `.learnings/`，影响阶段3（精读策略）
- **阶段6（模型更新）** 的置信度影响阶段7（评分）的权重
- **阶段8（写作触发）** 从阶段6（模型库）调取数据
- **阶段9（复现触发）** 从阶段3（data/精读/）和阶段6（models/）调取数据

## 质量阈值动态调整

质量阈值的联动规则：

| 条件 | 调整 | 影响 |
|------|------|------|
| 模型置信度 < 0.2 | 升级阅读深度（粗略→标准→精细） | 阶段1→阶段3 |
| 用户批注密度 > 70% | 升级分析粒度（段落→句子→子句） | 阶段3 |
| 连续3篇论文理解验证失败 | 降低模型更新置信度权重 | 阶段6 |
| 同类模型数量 ≥ 5 且置信度 ≥ 0.7 | 写作指导可同时引用多个模型 | 阶段8 |
