# 理解验证环节

## 核心定位

理解验证是模型更新的守门人。如果对论文的理解是错误的，基于此抽取的模型也会错。

验证通过 → 模型更新；验证未通过 → 报告主人重新阅读。

## 三项必做检验

### 检验一：转述检验

**通过标准**：能用自己的话向非专业人士说清楚核心论点。

**执行方式**：
1. 用一段话（不超过200字）转述论文的核心论证
2. 不使用论文中的术语（用日常语言替代）
3. 如果转述中出现"这里不太确定"或需要回头查原文 → 未通过

**判定**：能流畅转述 = 通过；卡顿/模糊 = 未通过。

### 检验二：反例检验

**通过标准**：能找到至少一个动摇论点的反例，并判断论文是否已处理。

**执行方式**：
1. 主动寻找论文论点的反例（即使论文没有提及）
2. 检查论文是否已经处理了这些反例
3. 如果论文未处理且反例确实动摇核心论点 → 标记为"存在未处理的质疑"

**判定**：论文已处理反例，或反例不构成实质性挑战 = 通过；存在未处理的有效反例 = 标记后通过（但置信度降级）。

### 检验三：衰减预判

**通过标准**：预估重读时是否会有新的理解冲击。

**执行方式**：
1. 自问："一周后重读这篇论文，我是否会有不同的理解？"
2. 如果有预感某些部分理解可能不完整 → 标记为"理解深度待确认"
3. 如果确信已完全理解 → 通过

**判定**：有新的理解冲击预期 = 通过（说明论文足够深刻）；毫无新意预期 = 未通过（可能理解停留在表面）。

## 三项判定汇总

| 检验 | 通过 | 未通过 | 处理 |
|------|------|--------|------|
| 转述检验 | 流畅转述 | 卡顿/模糊 | 未通过 → 不进入模型更新 |
| 反例检验 | 已处理/不构成挑战 | 存在未处理有效反例 | 标记后通过（置信度降0.1） |
| 衰减预判 | 有冲击预期 | 无新意预期 | 未通过 → 标记"需重读" |

## 与模型更新的关系

```
精读完成
    ↓
理解验证（三项检验）
    ↓
┌─ 全部通过 ──→ 进入模型更新（阶段6）
│
└─ 任一项未通过 ──→ 报告主人"理解验证未通过，需重新阅读"
                    ↓
              记录到 .learnings/{paper_id}.json 的 verification 字段
```

## 验证记录格式

```json
{
  "verification": {
    "paraphrase_test": {
      "passed": true,
      "paraphrase": "用日常语言转述的论文核心论点…"
    },
    "counterexample_test": {
      "passed": true,
      "counterexample_found": "找到了一个反例…",
      "paper_handled": true
    },
    "decay_prediction_test": {
      "passed": true,
      "expected_new_insight": "重读时可能会对…有更深理解"
    },
    "overall": "passed",
    "verified_at": "2026-06-20T12:00:00"
  }
}
```
