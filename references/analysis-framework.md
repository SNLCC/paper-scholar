# 100% 覆盖率精读框架（8 条研读规范）

## 总纲

精读不是"读懂论文内容"，而是**将论文的论证结构和写作方法拆解到可迁移的最小颗粒度**。

8 条规范分为两个层次：

```
规范 1-5：理解层（"这篇论文在说什么、怎么说"）
  段落拆解 → 框架提炼 → 引用标注 → 表格对比 → 双向归类

规范 6-8：反思层（"这篇论文的价值是什么、与我有什么关系"）
  批注还原 → 时代局限评估 → 多维评分与研究关联
```

**第一原则**：所有分析基于语义理解，不使用关键词/信号词判定逻辑关系。

---

## 规范 1：段落级别论证拆解

### 含义

不停留于"第一章第一节在讲什么"这种大节标题层面，要拆到**子论点、子流派的颗粒度**。

### 执行标准

每个子标题、每段、每句都要还原出"作者在这一层想说什么"。

| 层级 | 拆解标准 | 输出格式 |
|------|---------|---------|
| 章节 | 该章的实际修辞功能（不依赖标题判断） | `实际功能：建立研究合法性` |
| 段落 | 该段在论证链中承担的任务 | `论证任务：提出对立观点并反驳` |
| 句子 | 该句的修辞动作 | `修辞动作：让步承认既有贡献` |

### 示例

```
章节："Theoretical Framework"（标题）
  ├─ 实际功能：并非"理论框架"，而是"通过文献梳理识别缺口"
  │
  ├─ 段1：总结既有研究的两条路径
  │   └─ 功能：呈现学术现状，为缺口识别做铺垫
  │       ├─ 句1：路径A的核心主张
  │       ├─ 句2：路径A的贡献
  │       ├─ 句3：路径B的核心主张（与A形成对比）
  │       └─ 句4：路径A与B的共同局限
  │
  ├─ 段2：提出本文的理论整合方案
  │   └─ 功能：回应上文提出的局限
  │
  └─ 段3：推导研究假设
      └─ 功能：将理论框架操作化为可检验的命题
```

## 规范 2：分类框架提炼

### 含义

识别论文采用的分类框架——它如何对既有文献进行归类、划分、对比。

### 分析维度

| 维度 | 说明 | 示例 |
|------|------|------|
| **框架名称** | 给这个分类方式命名 | "技术采纳三阶段框架" |
| **分类维度** | 按什么标准划分 | "按时间顺序 / 按理论范式 / 按方法论立场" |
| **要素数量** | 分成几类 | 三类、四象限、二维矩阵 |
| **学科来源** | 框架来自哪个学科/理论传统 | "源于科学社会学" |
| **框架自洽性** | 分类是否一致，是否有交叉重叠 | "类别A和类别B在X维度上有交叉" |

### 执行要求

每篇论文精读时必须回答：

```
分类框架提炼卡
─────────────────────
框架名称：___________
分类维度：___________
要素数量：___________
学科来源：___________
自洽性评估：__________
对本文论证的作用：_____
```

## 规范 3：原文引用 + 位置标注

### 含义

关键引文逐字复制，标注页码和在论证链中的功能。

### 执行要求

**不允许**只写"作者认为范式很重要"这类模糊转述。

**必须**逐字摘录 + 标注页码 + 说明在论证链中扮演什么角色。

### 格式

```
[引文] "While prior research has extensively examined the direct effects of X on Y,
        the mechanisms through which X operates in the context of Z remain poorly understood."
[页码] p.3
[功能] 让步式缺口声明——承认既有贡献，指出研究空间，为本文的研究合法性奠基
[可迁移模板] "While [已有研究] has extensively examined [已知内容],
              the [本文关注点] in the context of [情境] remain[s] poorly understood."
```

### 选择标准

以下类型的内容必须引用标注：

| 类型 | 判断标准 |
|------|---------|
| 核心论点句 | 论文的核心 claim |
| 定义句 | 关键概念的界定 |
| 转折句 | 论证方向的转换点 |
| 结论句 | 章节或全文的收束 |
| 标志性引用 | 对奠基性文献的引用方式 |

## 规范 4：表格化对比

### 含义

并列内容（流派、转向、论点）用表格对比，不用纯文字叙述。

### 执行要求

**凡涉及 ≥3 个并列项，强制使用表格。**

### 示例

| 维度 | 流派A | 流派B | 流派C |
|------|-------|-------|-------|
| 核心假设 | X决定Y | X受Y影响 | X与Y互构 |
| 方法论偏好 | 量化分析 | 质性研究 | 混合方法 |
| 代表人物 | Smith(2010) | Jones(2012) | Lee(2015) |
| 本文立场 | 批评 | 部分借鉴 | 主要采纳 |

### 适用场景

- 文献综述中的流派对比
- 理论框架中的概念对比
- 研究发现中的案例对比
- 讨论中的解释路径对比

## 规范 5：双向归类

### 含义

每个论点同时做**正向**（来源脉络）和**反向**（否定前人）分析。

### 分析框架

```
论点X："技术采纳受组织文化影响"
    │
    ├─ 正向溯源（来源脉络）
    │   这个论点建立在谁的研究之上？
    │   → 继承 Rogers(2003) 的创新扩散理论
    │   → 发展自 Davis(1989) 的技术接受模型
    │   → 引入组织文化变量（新的理论贡献）
    │
    └─ 反向定位（否定前人）
         这个论点否定了什么？
         → 否定"技术采纳是纯技术问题"的假设
         → 挑战 Smith(2015) 的"理性采纳模型"
         → 与 Jones(2018) 的"制度压力论"形成竞争
```

### 执行要求

每篇论文的每个核心论点都需要做双向归类。

输出格式：

```
[论点] ____________________
  正向（继承/发展）：→ ______
  反向（否定/挑战）：→ ______
```

## 规范 6：批注逻辑还原

### 含义

从用户批注的顺序和注释内容，反推用户阅读时的认知路径。输出用户的"理解轨迹图"。

### 分析维度

```
用户批注轨迹（按页码顺序）
    │
    ├─ p.3 高亮 "机制尚不明确" + 评注 "core gap"
    │   → 用户在此处识别到了论文的核心缺口声明
    │
    ├─ p.5 高亮 "本研究采用..." + 评注 "DID方法"
    │   → 用户关注方法论选择
    │
    ├─ p.8 红色高亮 "假设3" + 评注 "存疑"
    │   → 用户对假设3的推导有疑问
    │
    └─ p.12 高亮 "与 Smith(2020) 一致" + 评注 "参见我的文献"
        → 用户建立了与自身研究的关联
    │
    ▼
用户理解轨迹图：
    识别核心问题 → 关注方法 → 质疑具体假设 → 关联自身研究
```

### 理解轨迹图的输出格式

```json
{
  "paper_id": "ABC123",
  "cognitive_path": [
    {"stage": 1, "page": 3, "action": "识别核心问题", "trigger": "gap statement"},
    {"stage": 2, "page": 5, "action": "关注方法", "trigger": "method description"},
    {"stage": 3, "page": 8, "action": "批判性质疑", "trigger": "hypothesis derivation"},
    {"stage": 4, "page": 12, "action": "关联自身", "trigger": "citation of related work"}
  ],
  "reading_depth": "批判性阅读",
  "focus_characteristics": "重视方法论和假设推导"
}
```

### 执行前提

仅当用户有 Zotero 批注时执行此规范。无批注时跳过。

## 规范 7：时代局限评估

### 含义

明确写出论文的时间局限和后续发展。用今天的视角审视一篇论文，对其时效性做出判断。

### 分析维度

| 维度 | 说明 |
|------|------|
| **数据时效性** | 论文使用的数据是否已过时？如果有更新数据，结论是否会变化？ |
| **理论时效性** | 论文的理论框架在当下是否仍被接受？是否有后续理论修正或颠覆？ |
| **方法时效性** | 论文采用的方法是否已被更优方法取代？ |
| **事件依赖** | 论文的结论是否依赖于特定历史时期或事件？ |
| **后续发展** | 该论文发表后，领域内有哪些重要发展？ |

### 输出格式

```
时代局限评估
─────────────────────
发表年份：2018
评估时间：2025

数据时效性：⭐⭐⭐ 基于2015年问卷数据，在当下可能部分过时
理论时效性：⭐⭐⭐⭐ 核心理论框架仍被广泛使用，但已有3个修正版本
方法时效性：⭐⭐⭐⭐ 方法论仍有效，但新兴的数据分析方法可以提供更丰富的洞见
事件依赖：⭐⭐ 不依赖特定历史事件

整体评价：这篇论文的核心论证仍然有效，但引用时需注意其数据年代
```

## 规范 8：多维评分 + 研究关联

### 三维度评分

| 维度 | 评分 | 说明 |
|------|------|------|
| 选题质量 | ⭐⭐⭐⭐⭐ | 与研究相关性 / 时效性 / 学科贡献度 |
| 论证质量 | ⭐⭐⭐⭐ | 论证链条完整性 / 概念准确性 / 反驳处理 |
| 写作质量 | ⭐⭐⭐⭐⭐ | 结构规范性 / 语言清晰度 / 信息密度 |

评分基于精读后的综合判断，不是简单打分，每项需附文字说明。

### 研究关联（必填三项）

每篇论文分析完成后，必须做三个方向的研究关联分析：

**支持：** 论文的哪些观点/框架支持用户的研究方向？
- 框架 X 可直接借鉴到用户的研究中
- 方法 Y 适用于用户的研究对象

**反驳/质疑：** 论文的哪些观点与用户立场有张力？
- 论文主张 A，但用户的核心立场是 B
- 论文的方法论预设与用户的理论路径冲突

**补充：** 论文提供了哪些用户尚未关注的视角/文献？
- 提出作者 D（用户尚未关注）
- 提供视角 E（可纳入思考）
- 引文 F（可进一步追踪）

## 精读产出规范

每篇论文精读完成后，必须产出：

### 1. 精读报告（.md）

覆盖全部 8 条规范，包含：
- 论文元数据（标题、作者、期刊、年份）
- 三段式评分（规范7、规范8）
- 逐章逐段分析（规范1、规范2、规范4、规范5）
- 关键引文标注（规范3）
- 批注轨迹图（规范6，如有批注）
- 研究关联分析（规范8）

### 2. 结构化数据（.json）

供脚本处理，格式与 `assets/model-template.json` 兼容。
