# 复现触发

## 核心理念

不仅要形成模型，还要对已读论文本身的框架内容有所提炼。
能从几个月前的精读笔记中立刻提取出整张骨架。

## 触发条件（四种之一）

| 触发条件 | 示例 | 优先级 |
|---------|------|--------|
| **主人提及** | "上次那篇关于XX的论文…" | 高 |
| **写作需要** | 写到相关章节时自动提示 | 中 |
| **时间提醒** | heartbeat 主动推送（定期回顾） | 低 |
| **研究阶段推进** | 论文进入下一阶段需要回顾文献 | 中 |

## 复现内容

```
触发"复现"某篇论文时，从以下来源提取信息：

┌─ data/精读/{paper_id}/ ─┐
│  核心论点摘要              │
│  论证结构图               │
│  原文引用 + 位置标记       │
│  主人批注的论证线索        │
└──────────────────────┘
         +
┌─ models/{model_id}.json ─┐
│  该论文所属类型的模型特征   │
│  同类论文的共同规律        │
│  置信度评估               │
└──────────────────────┘
```

## 复现输出格式

```markdown
## 复现：{论文标题}

**核心论点**（一段话）：
…

**论证结构**：
第一节（功能）：…
第二节（功能）：…
第三节（功能）：…
第四节（核心论点）：…
结论（意义升华）：…

**关键引用**：
> "…"（p.3, para 2）

**主人批注线索**：
- 批注关注焦点：理论框架（35%）和论证分析（25%）
- 批注粒度：中等（每段1-2句）
- 关键质疑："…"

**同类论文模型参考**（置信度 0.72）：
- 该类型论文的典型结构：…
- 常见论证步骤：…
```

## 与"复现论文"的区别

| | 此处的"复现" | SKILL.md 中的"reproduce" |
|---|---|---|
| **用途** | 主人提问时从记忆浮现笔记 | 分析新论文时复用模型作为参考 |
| **数据源** | data/ 中的精读笔记 | models/ 中的模型 |
| **触发方式** | 被动（主人问） | 主动（写作用到） |
| **输出** | 整张论文骨架 | 写作建议 |
